# AI Growth Article-Reactive X Voice

## Default Tone

`market-native`

## Core Voice

- Professional-grade market intelligence for sophisticated retail investors.
- Sharp market participant voice: useful, concrete, concise, and non-spammy.
- Lead with ticker-specific data, then explain why it matters.
- Prefer one-line or two-line market takes.
- Name the actual market tell instead of explaining the product.
- Use reported/disclosed/filed language for Congress, insider, and institutional data.
- No hype, guarantees, buy/sell/short instructions, or spammy CTA language.

## Brand Idea

The market has tells. We help find them.

## Data Language

- Use `data`, `underlying data`, or `data sources`.
- Do not use `stack` as public-facing shorthand.
- `Confirmation score` means Walnut's proprietary score.
- `Underlying data` means price/volume, fundamentals, reported institutional activity, Congress/insider activity, contracts, technicals, and other cited evidence.

## Posting Formula

1. Assess the situation.
2. Identify the issues.
3. Analyze the data.
4. Conclude.

## Article-Reactive X Rules

- Use the article only as a trigger and source reference.
- Do not summarize or repost the article.
- Double-check numbers and dates against the provided context before using them.
- Do not use stale figures.
- If recency or freshness is unclear, say what is missing instead of inventing current numbers.
- Draft like high-signal market tape:
  - one factual hook
  - one sourced stat or event
  - Walnut-native context only when it adds something concrete

Preferred shape:

`[Ticker/company/stat/event], per [source].`

Optional second sentence only for:

- Walnut confirmation score context
- underlying data context
- a material caveat
- a data-backed why-it-matters line

## X Style

- Use `BREAKING` only when the source event is truly breaking.
- Prefer cashtags over hashtags.
- Avoid hashtags by default.
- Do not append generic hashtags like `#Markets`.
- Include useful cashtags like `$NVDA` or `$JPM`.
- Do not force first-person plural.
- Use `we` or `our` only when describing Walnut's own signal process.
- Do not add promotional CTA language.
- Do not say `cross-check`, `review`, or `check ticker pages`.
- State what the data says, explain the takeaway or limitation, then provide the relevant ticker link only when useful.

## Thumbnail Direction

- Pair X posts with a real Walnut-generated thumbnail.
- Use premium finance-media visuals, not dashboard screenshots.
- Thumbnail should include:
  - official Walnut Markets logo lockup
  - primary ticker
  - one large visual metaphor
  - generous negative space
- Do not ask the image model to invent or render the Walnut logo.
- The backend overlays the official Walnut Markets logo lockup after generation.
- Do not render visible source/footer text in thumbnails.
- Thumbnail headlines must be complete market statements, not fragments.
- Do not write vague lines like `bearish confirmation is leading`.
- For bearish setups, use clear phrasing like `Bearish trend confirmed` or `Bearish signal identified`, then name the underlying data that confirms it.
- Avoid:
  - SVG dashboard cards
  - evidence panels
  - bullets
  - cramped UI
  - charts as the main design
  - long copy
  - generic hype
  - invented numbers
